# Protocol Guide Instagram Strategy

> Visual storytelling for the EMS community. Build brand as part of the EMS ecosystem.

**Last Updated**: January 2026
**Platforms**: Instagram (Primary), TikTok (Secondary sync)
**Handle**: @protocolguideapp

---

## Executive Summary

Protocol Guide's Instagram presence positions the app as an authentic member of the EMS community first, product second. Visual storytelling celebrates EMS professionals, educates through shareable content, and drives organic app awareness through community validation rather than direct selling.

**Core Positioning**: "Built by EMS, for EMS"

**6-Month Goals**:
- 10,000 followers (organic growth)
- 4%+ engagement rate (vs 1.5% industry average)
- 500 UGC submissions via #MyProtocolGuideStory
- 50 influencer/department partnerships
- Trackable contribution to 2,000 app downloads

## Visual Identity Guidelines

### Brand Colors

```css
/* Primary Palette */
--ems-red: #C41E3A;        /* Primary accent, emergency-themed CTAs */
--protocol-blue: #2563EB;  /* App brand, educational content */
--safety-orange: #F97316;  /* Alerts, important highlights */

/* Secondary Palette */
--medical-white: #FFFFFF;  /* Backgrounds, clean layouts */
--charcoal: #1F2937;       /* Text, dark mode backgrounds */
--muted-gray: #6B7280;     /* Secondary text, captions */
```

### Typography

```
Headlines: Inter Bold (or system -apple-system)
Body: Inter Regular
Accent: Mono for code/dosages (SF Mono, Consolas)
```

### Visual Style Standards

| Element | Guideline |
|---------|-----------|
| **Photos** | Authentic, well-lit, minimal filters. No heavy editing. |
| **Graphics** | Clean, readable, dark-mode friendly. High contrast. |
| **Screenshots** | Device mockups with consistent framing. |
| **Video** | 9:16 for Reels, professional but approachable. |
| **Icons** | Simple line icons, consistent stroke width. |

### Filter/Preset

Use consistent editing for all photos:
- Slight contrast boost (+10)
- Subtle warmth adjustment for skin tones
- Keep original lighting character
- Never over-saturate

## Content Pillars (Detailed)

### Pillar 1: Community Celebration (35%)

Celebrate the humans behind the badge. This builds emotional connection and generates UGC.

#### Content Types

**1. Medic Spotlight (Weekly)**
Feature individual EMS professionals with their permission.

Format: Carousel (4-5 slides)
- Slide 1: Portrait photo with name/role overlay
- Slide 2: "Why I became a medic" quote
- Slide 3: Favorite call type or specialty
- Slide 4: Advice for new providers
- Slide 5: Protocol Guide CTA (subtle)

Caption Template:
```
Meet [Name], a [Role] from [Location].

"[Quote about their journey or why they do this work.]"

What drew you to EMS? Drop your story below.

#ProtocolGuide #EMSLife #[State]EMS #MedicSpotlight
```

**2. Department Feature (Bi-weekly)**
Showcase departments using Protocol Guide.

Format: Single image or carousel
- Department group photo (with permission)
- Apparatus/equipment shots
- Protocol Guide in action on their devices
- Quote from chief/training officer

**3. First Responder Friday**
Weekly celebration post every Friday.

Format: Single image with text overlay
- Quote or stat about EMS
- Appreciation messaging
- Community tag encouragement

Example Copy:
```
Another Friday, another 24.

To every medic pulling a shift this weekend: we see you.

Tag someone you'd want as your partner on a tough call.

#FirstResponderFriday #EMSLife #ProtocolGuide
```

**4. Milestone Celebrations**
User-submitted achievements.

Types:
- New certifications (NREMT, CCP, FP-C)
- Retirements
- Service anniversaries
- First code save
- Promotions

**5. User Testimonials**
Real feedback from Protocol Guide users.

Format: Quote graphic + app screenshot
- Clean design with user quote
- Optional: before/after scenario
- Always get written permission

### Pillar 2: Educational Content (30%)

Position Protocol Guide as a trusted educational resource. High-save, high-share content.

#### Content Types

**1. Protocol of the Week (Weekly Carousel)**

Format: 5-7 slide carousel (1080x1350)
- Slide 1: Protocol title with visual hook

- Slide 6: Common mistakes/pearls
- Slide 7: "Save this for your next shift" + Protocol Guide CTA

- Pediatric Assessment Triangle
- Stroke Recognition: BE-FAST
- Pain Management Decision Tree
- Allergic Reaction vs Anaphylaxis
- RSI Checklist
- OB Emergencies Quick Reference

Visual Requirements:

- Large, readable text (minimum 24pt)
- Clear hierarchy with icons

**4. Did You Know? (Weekly Stories)**

**5. Quiz Posts (Stories - Daily)**

Interactive Story quizzes:

## Compliance & Safety Guidelines

### Medical Content Review Process

1. **Creation**: Draft by content team
2. **Medical Review**: Licensed paramedic/physician review
3. **Source Check**: Verify citations (AHA, NAEMSP, etc.)
4. **Disclaimer Addition**: Standard disclaimer included
5. **Legal Review**: If needed for sensitive topics
6. **Publish**: With audit trail

**Standard Disclaimer (Educational Posts)**:
```
This content is for educational purposes only and does not
constitute medical advice. Always follow your local protocols
and medical director's guidance.
```

### Patient Privacy (Absolute)

- **NEVER** post content showing actual patients
- **NEVER** include patient identifiers of any kind
- Training/simulation photos must use mannequins
- Staged scenarios clearly labeled as "training scenario"
- User submissions screened for HIPAA compliance

### Content Boundaries

**Always Avoid**:
- Political content of any kind
- Controversial takes on EMS issues
- Criticism of specific agencies/protocols
- Graphic medical imagery (blood, trauma)
- Patient-related humor
- Disparaging comments about EMS roles

**Crisis Protocol**:
- Pause scheduled content during national tragedies
- Supportive messaging only during EMS incidents
- No exploitation of current events
- Sensitivity review before resuming normal content

### FTC Compliance

**Paid/Sponsored Content**:
- Clear disclosure: #ad or #sponsored
- Above the fold (not buried in hashtags)
- Influencer contracts include compliance requirement

**User-Generated Content**:
- Get written permission before reposting
- Credit original creator with tag
- Archive permission documentation
